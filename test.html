Comparison of SJS, SRTF and FCFS

Problem of Starvation

Take an example of 3 processes A and B and we are using SJF scheduling algorithm, if A has a execution time of 2ns and B has a execution time of 10ns, then A has 1st priority and B has 2nd priority according to SJF this is the scenario.

Now consider, just before the completion of A, a process C arrives and has the execution time of 3ns then after completion of A, C will be given priority, just before the completion of C consider a new process arrives and has a execution time of 2ns then obviously it would be scheduled before A.

Like this there can be many processes coming to RAM and which can have a better priority than B and there are quite a good chance that B has to "wait indefinately" and this problem is called Starvation and here B is starving.

Comparision with different Algorithms :

Every priority based algorithm will suffer from Starvation Problem because we never get any defination of which process will be scheduled next and which process has to wait, so there are better chances of indefinite waiting time.

First Come First Serve Algorithm :

Starvation doesn't fall into the scope of FCFS, because in case of FCFS, everything is clear, no process need to wait indefinitely, if a process need to wait then also that would be definate because the CPU knows which process came n ext and thus take it without any indefinate behaviour.

Convoy Effect :

How inefficient it would be, if you have to stand for 1 hour behind the person who has 100 items to bill and you just have 1 item to bill, actually this is Convoy effect.

When a small process has to wait for a big process to complete such a situation is called convoy effect, obviously a disadvantage.

Comparison with Algorithm :

FCFS Scheduling Algorithm :

This is one of the prime problem of FCFS, this is a fixed way with no flexibility over time for a shorter process to take up which came after the bigger process, it has to wait wait and wait till the first process complete which can take huge time

SJF Scheduling Algorithm :

So far its a priority based algorithm, you may say that it will not face any convoy problem, but its not true, SJF will take the shortest job first which has arrived in that point of time, once at t=0 a 1000 ns process schedules then there is no scope of 1ns process to be scheduled arriving at t = 1, hence SJF also faces convoy effect.

SRTF Scheduling Algorithm :

SRTF has a premitive workflow hence it is saved from a problem of convoy effect. Once scheduled a 100ns process if a 1 ns process comes it will switch the context and give priority to 1ns process.

Practical Implementation of Algos

If we compare SJF, SRTF and FCFS then for sure SJF and SRTF gives a better throughput ( process/time-unit ) but their implementation is very tough, because we need to knwo the burst time of every process which is somewhat very tough but when we talk of FCFS then we can just set a clock in the long term schedulers and get to know which process has arrived first and then implement the algo.
